使用PicGo+Github快速部署个人图床

一、简介

1. PicGo是什么

PicGo是一款图床工具，支持微博图床，七牛云，腾讯云COS，又拍云，GitHub，阿里云OSS，SM.MS，Imgur 等8种常用图床，简单易用。

  • 简单易用
  • 复制Markdown链接
  • 支持的图床

2. 公共图床推荐

  • 微博图床 : 封装成chrome插件，比较方便
  • imgur
  • sm.ms
  • 聚合图床
  • 路过图床 : 不允许使用梯子
  • 。。。

3. 生成 token

在 GitHub 上生成一个 token 以便 PicGo 操作我们的仓库，来到个人中心，选择 Developer settings 就能看到 Personal access tokens，我们在这里创建需要的 token。

  • 建议把token记好

4. 设置PicGo

  • 设定仓库名的时候，是按照“账户名/仓库名的格式填写”，不能带空格
  • 分支名填写master即可
  • 将刚才的token贴在设定Token
  • 指定储存路径可不写，写了的话会在repository下创建一个与之对应的文件夹
  • 自定义域名的作用是，在上传图片后成功后，PicGo会生成“自定义域名+上传的图片名”形式的访问链接，将其放到剪切板。这是格式： https://raw.githubusercontent.com/用户名/项目名/分支名
  • 一般可以不写自定义域名

5. 其他说明

  1. 建议设置一下热键
  2. 这个工具的作用是把本地图片推到仓库去，默认不同步删除操作。即
    • 将PicGo相册的图片删除后，仓库里的图片不会咋样；
    • 直接把仓库删了,PicGo相册里的图片也暂时不会消失。当然，链接会挂。
  3. PicGo支持扩展
    • githubPlus插件地址
  4. 如果遇到这种无法显示的问题：
    • 图片外链能够正常打开
    • 应用程序的相册无法显示
    • 点击单张图片，稍加加载之后无法显示

      …大概率只是网不好…毕竟国内直连Github也不是很稳定，挂个代理试试
